Features are built spec-first. The loop starts the moment a [Linear issue](/method/issues) is handed to an agent in Cursor: the agent runs the [spec-kit](https://github.com/github/spec-kit) loop, every artifact lands in `specs/NNN-name/`, and it comes back to you at three points.

## Your job

<Steps>
  <Step title="Describe">
    Hand the agent the [issue](/method/issues). It runs `/speckit.specify` first, required by the [contract](/stack/tooling), and returns `spec.md`. Done when the scenarios cover what you meant, nothing invented.
  </Step>
  <Step title="Answer">
    Answer the agent's clarify questions, then review the plan against the [constitution](/architecture/structure). Done when no `[NEEDS CLARIFICATION]` remains and nothing you did not ask for slipped in. Agents are over-eager, catching that is your job.
  </Step>
  <Step title="Review">
    Review the PR. Done when every scenario test is green and converge leftovers are filed as new issues.
  </Step>
</Steps>

## Path

Features take the full loop above. Fixes and tweaks take the light path: issue, branch, fix, PR, existing scenarios cover the regression.

Done is not the agent saying done, done is the gate, see [Testing](/method/testing).
