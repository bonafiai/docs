title: "Commands"
description: "The loop from your side: what you say, what you review."
---

The agent executes every `/speckit.*` command. You feed it intent, answer its questions, and review its artifacts. Three touchpoints, everything else is automated. This page is your checklist per step, see [spec-kit](https://github.com/github/spec-kit) for the tool itself.

## Constitution

Runs once per repo, then amended, never rewritten per feature.

- **Say**: the principles, or point it at the architecture pages.
- **Agent**: writes `.specify/memory/constitution.md`, every later plan is gated on it.
- **Review**: the rules match the architecture pages, nothing invented.

## Specify

- **Say**: what and why, no tech stack. From a Linear issue, in behavior terms:

```text
/speckit.specify Reviewers approve or reject proposed values from parsed
statements. Uncertain mappings queue for review, approved proposals become
truth. A reviewer sees why each value was proposed.
```

- **Agent**: creates the branch and `specs/NNN-name/spec.md` with user stories and acceptance scenarios.
- **Review**: the scenarios cover the behavior you meant, nothing invented. This is the cheapest moment to catch drift.

## Clarify

- **Say**: answers. The agent asks coverage-based questions before planning, skipping this is an explicit decision, not a default.
- **Agent**: records every answer in the spec's Clarifications section.
- **Review**: your answers landed, no `[NEEDS CLARIFICATION]` markers remain.

## Plan

- **Say**: tech constraints beyond the constitution, if any:

```text
/speckit.plan Values land via the review module's commit path. Parsing
stays behind the parser seam. No new tables outside the review module.
```

- **Agent**: writes `plan.md`, `research.md`, `data-model.md`, and `contracts/`, checked against the constitution and its gates.
- **Review**: gate results, and over-engineering. Agents are over-eager, catching added components nobody asked for is a human job.

## Tasks

- **Say**: nothing, the agent chains it.
- **Agent**: writes `tasks.md`, ordered by dependency, parallel tasks marked `[P]`.
- **Review**: the order respects dependencies, no task exists that the spec does not demand.

## Analyze

- **Say**: nothing, the agent chains it.
- **Agent**: cross-checks spec, plan, and tasks for consistency and coverage before implementation.
- **Review**: the findings. Gaps found here cost minutes, gaps found after implement cost days.

## Implement

- **Say**: nothing, the agent chains it. It refuses to run without spec, plan, and tasks, that refusal is the feature.
- **Agent**: executes the tasks, code and tests.
- **Review**: the PR, with every scenario test green, see [Testing](/method/testing).

## Converge

- **Say**: nothing, run after shipping.
- **Agent**: diffs spec against code, appends remaining work.
- **Review**: leftovers became Linear issues, nothing silently dropped.
