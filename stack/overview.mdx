We pick the best and fastest tool per job, each behind a seam we can swap. This is the inventory, the in-depth implementation pages per SDK come later. Extend deliberately.

## Product

| Name | Description |
| --- | --- |
| [Next.js 16](https://nextjs.org/docs) | App router, RSC for the review UI, Vercel deployment |
| [Hono](https://hono.dev/docs) | Typed API on a catch-all route, modular routers per domain |
| [WorkOS](https://workos.com/docs) | Identity and authorization: AuthKit sessions, FGA for object-level access |
| [Drizzle](https://orm.drizzle.team/docs/overview) + [PlanetScale](https://planetscale.com/docs) | Typed SQL, migrations, Postgres dev branches that mirror prod |
| [Zod](https://zod.dev) | Runtime validation and type inference at every boundary |
| [Inngest](https://www.inngest.com/docs) | Typed events, durable steps, crons, one publish contract for everything async |
| [Vercel AI SDK](https://ai-sdk.dev/docs) | The agent loop: streaming, tool calling, structured output |
| [Vercel AI Gateway](https://vercel.com/docs/ai-gateway) | One attribution path, multi-model routing, BYOK |
| [Pipedream Connect](https://pipedream.com/docs/connect) | The connectors layer: third-party accounts (bank, email, drive), org-scoped |
| [Cloudflare R2](https://developers.cloudflare.com/r2/) | Object storage, content-addressed keys, EU-capable |
| [LlamaParse](https://docs.cloud.llamaindex.ai/llamaparse/getting_started) | The parser behind the parser seam, swappable in no time (Reducto under evaluation) |
| [Vercel](https://vercel.com/docs) | Hosting, preview deploys, edge where it helps |
