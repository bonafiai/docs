The stack, each choice and what it does for us. Wiring pages flow in later from the product core.

## Product stack

| Choice | Why |
| --- | --- |
| Next.js 16 | App router, RSC for the review UI, Vercel deployment |
| Hono | Typed API on a catch-all route, modular routers from day one |
| WorkOS | Our identity and authorization platform: AuthKit sessions, FGA for object-level access |
| Drizzle + PlanetScale Postgres | Typed SQL, migrations, dev branches that mirror prod |
| Inngest | Typed events, durable steps, crons; one publish contract for everything async |
| Vercel AI SDK v6 | The agent loop: streaming, tool calling, structured output |
| Vercel AI Gateway | One attribution path, multi-model routing, BYOK |
| Cloudflare R2 | Object storage, content-addressed keys, EU-capable |
| LlamaParse v2 | The parser, producing one frozen artifact per source behind the parser seam |
| Vercel | Hosting, preview deploys, edge where it helps |

## Toolchain

| Choice | Why |
| --- | --- |
| Bun | Runtime and package manager in one tool |
| oxlint + prettier | Fast lint, consistent format, invariant scripts ride along |
| Vitest | Unit and integration tests |
| Playwright | E2E acceptance tests bound to spec scenarios |
| spec-kit | Constitution, specify, plan, tasks, implement, converge |
