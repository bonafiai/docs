We pick the best and fastest tool per job, each behind a seam we can swap. Wiring pages flow in later from the product core.

## Product

| Name | Description |
| --- | --- |
| Next.js 16 | App router, RSC for the review UI, Vercel deployment |
| Hono | Typed API on a catch-all route, modular routers from day one |
| WorkOS | Identity and authorization: AuthKit sessions, FGA for object-level access |
| Drizzle + PlanetScale | Typed SQL, migrations, Postgres dev branches that mirror prod |
| Inngest | Typed events, durable steps, crons, one publish contract for everything async |
| Vercel AI SDK v6 | The agent loop: streaming, tool calling, structured output |
| Vercel AI Gateway | One attribution path, multi-model routing, BYOK |
| Cloudflare R2 | Object storage, content-addressed keys, EU-capable |
| LlamaParse v2 | The parser behind the parser seam, swappable in no time (Reducto under evaluation) |
| Vercel | Hosting, preview deploys, edge where it helps |
