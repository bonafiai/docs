Tooling is what every teammate and agent gets by opening the repo. Rosters live here and in committed config, nothing ad-hoc on individual machines.

## MCP servers

Configured in [`.cursor/mcp.json`](https://github.com/bonafiai/docs/blob/main/.cursor/mcp.json).

| Server | Purpose | Auth |
| --- | --- | --- |
| Mintlify | Read and search this docs hub from agents | OAuth (browser) |
| Linear | Create, read, and update issues from agents | OAuth (browser) |

New MCP servers, skills, and dependencies are agreed in Slack or Linear and land here via PR.

## Skills

Official vendor sources only, pinned via `skills-lock.json`. No low-star community skills.

| Skill | Source |
| --- | --- |
| Mintlify | `https://mintlify.com/docs` |

Install: `npx skills add <source>`.

## Core dependencies (code repo)

| Dependency | Role |
| --- | --- |
| `bun` | Runtime and package manager |
| `next` | App framework |
| `hono` | HTTP API |
| `drizzle-orm` | Database access |
| `inngest` | Event bus and jobs |
| `ai` + `@ai-sdk/*` | LLM calls via the gateway |
| `zod` | Runtime validation |
| `vitest` | Unit and integration tests |
| `@playwright/test` | E2E acceptance tests |

This is the day-0 posture, not every transitive dependency. Extend deliberately.

## Agent contract (code repo)

- No feature work without `specs/NNN-*/spec.md`
- First agent action on a missing spec: `/speckit.specify`
- Rules mirror: `.specify/memory/constitution.md` matches docs [Rules](/concept/rules)
