One app, one deploy, hard module boundaries. Every feature lands in the same shape, so anyone (human or agent) knows where things live without asking. The tree is the recipe, the same shape for every module:

```text
app/                    routes only, no business logic
  (app)/                route group, the authenticated product
    <feature>/
      page.tsx
      _components/      UI private to this route
      _lib/             helpers private to this route
  api/
    [[...route]]/       one catch-all, Hono owns /api
lib/
  <module>/             one folder per domain, same shape every time
    router.ts           its Hono routes, mounted by the catch-all
    schema.ts           its Drizzle tables, types derived, not written
    types.ts            hand-written domain types, only if not derivable
    jobs/               its Inngest functions, one file per job, crons included
    <domain>.ts         its domain logic, the fat part
  shared/               events registry, tenant db, gateway client
proxy.ts                the edge session gate
specs/                  feature specs from the loop
.specify/               spec-kit memory, the constitution
```

## Modules

A domain folder in `lib/` owns everything for that domain: its logic, its Hono router, its Drizzle tables, its jobs, and its types. New domain, new folder, same shape, per [Hono best practices](https://hono.dev/docs/guides/best-practices): modular sub-routers over one giant app. Every [Inngest function](https://www.inngest.com/docs/features/inngest-functions) lives in the owning module's `jobs/`, one file per job, a thin shell delegating to the domain logic next to it, and a cron is just a job with a [cron trigger](https://www.inngest.com/docs/guides/scheduled-functions), same folder, named for its sweep.

## App

`app/` is routing, nothing else. [Route groups](https://nextjs.org/docs/app/getting-started/project-structure) organize without touching URLs, `_components` and `_lib` colocate route-private code out of the router, and one catch-all hands `/api` to Hono, see [Hono on Vercel](https://hono.dev/docs/getting-started/vercel).

## Boundaries

Modules talk through exported functions and events, never another module's tables. `shared/` has three residents, the event registry, the tenant transaction, and the gateway client, and it stays that small on purpose: growth means something is misplaced. A type two modules need moves there, and that is a smell to watch.
