Apps route, `lib/` owns domain logic, `packages/ui` shares UI. Each app is self-contained: own `proxy.ts`, own auth boundary. Same template everywhere, placeholders show the shape. Guardrails are transcribed into `.specify/memory/constitution.md` in bonafiai/web.

## Apps

```text
apps/
  web/                              product app, default in microfrontends
    app/
    proxy.ts                        edge session gate, this app only
    microfrontends.json             local: /test → test app
  test/                             internal test app
    app/
    proxy.ts                        own gate, own auth boundary
packages/
  ui/
    src/components/                 shadcn CLI installs here
specs/                              spec-kit artifacts per feature
.specify/memory/constitution.md     agent contract, transcribed from this site
```

Locally, [microfrontends](https://turborepo.dev/docs/guides/microfrontends) composes both apps on one URL; production routing is separate. `apps/web` is the full product app at `/` in production, not a UI fragment.

## App

```text
app/                    routes only, no business logic
  (app)/
    <feature>/
      page.tsx
      _components/
      _lib/
  api/
    [[...route]]/       one catch-all, Hono owns /api
    inngest/            Inngest serve handler, web app only
proxy.ts                at app root, gates this app only
```

Route-private UI uses `_components` and `_lib`, shared UI from `@repo/ui/components` ([shadcn monorepo](https://ui.shadcn.com/docs/monorepo)). One catch-all hands `/api` to Hono routers in `lib/`.

## Lib

```text
lib/
  <module>/                         same shape for every domain module
    router.ts                       Hono routes, mounted by app catch-all
    schema.ts                       Drizzle tables; types derived, never hand-written
    types.ts                        hand-written only when not derivable
    jobs/
      <job>.ts                      one file per job, thin Inngest shell; crons live here too
    <module>.ts                     domain logic, the fat part
  shared/
    events.ts                       zod event registry, noun.verb names
    db.ts                           tenant-scoped Drizzle client
    inngest.ts                      Inngest client
    gateway.ts                      shared infra only; grows slowly
```

The foundation ships `lib/shared/` only. Domain modules arrive via approved specs.

## Rules

| Name | Description |
| --- | --- |
| Routes | `app/` routes only, no business logic |
| API | One catch-all per app; Hono routers live in `lib/<module>/router.ts` |
| Module shape | Every domain folder: `router.ts`, `schema.ts`, `types.ts`, `jobs/`, `<module>.ts` |
| Ownership | A module owns its tables, logic, jobs, and types |
| Boundaries | Modules talk through exported functions and events, never another module's tables |
| `shared/` | Events registry, tenant db, Inngest client, gateway client only; a type two modules need moves here, and that is a smell |
| UI | Route-private code in `_components` and `_lib`; shared components from `@repo/ui` |
| Specs | No feature work without `specs/NNN-*/spec.md`; no new `lib/<module>/` without an approved spec |

Placement rules: [Types](/architecture/types). Async rules: [Jobs](/architecture/jobs). Persistence rules: [Database](/architecture/database).
