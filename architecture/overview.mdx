Bonafi is a modular monolith: one app, one deploy, hard module boundaries. This page is the map. Each plane gets its own concise page, starting with [Structure](/architecture/structure).

| Name | Description |
| --- | --- |
| [Structure](/architecture/structure) | Domain modules in `lib/`, each owns its router, jobs, and tables |
| Request | Browser to response through one path, slow work publishes to the bus |
| Access | Session, tenant, permission, object. The edge never authorizes |
| Data | Tenant transactions, module-owned tables, truth through the commit path |
| Events | One typed publish contract, replayable at every stage |
| Intelligence | Every model call through the gateway, tools in one registry |
| Observability | One logger, request id from edge to job |

This is the kitchen. [Structure](/architecture/structure) says where everything lives, the other planes get their pages as the codebase lands. Feature specs are the recipes.
