Get the repos open in Cursor and the tooling provisioned. This is the fast lane — no theory required.

## Prerequisites

Before you begin:

- [Cursor](https://cursor.com) installed
- [Bun](https://bun.sh) 1.2+ (runtime and package manager for `mono`)
- [Mintlify CLI](https://www.npmjs.com/package/mint) for local docs preview (`npm i -g mint`)
- Access to the `bonafiai` GitHub org

## Get started

<Steps>
  <Step title="Clone the repos">
    Two repos, two roles:

    ```bash
    git clone https://github.com/bonafiai/docs.git
    git clone https://github.com/bonafiai/mono.git
    ```

    - **docs** — this spec hub (beliefs, rules, stack, decisions)
    - **mono** — the application codebase (reference and implementation)
  </Step>
  <Step title="Open in Cursor">
    Open the **docs** repo (or a multi-root workspace with both). Tooling auto-provisions from the repo:

    - `.cursor/mcp.json` — Mintlify and Linear MCP servers
    - Pinned skills via `skills-lock.json` (after install)
    - `AGENTS.md` — writing and agent contract for this repo
  </Step>
  <Step title="Link Linear with Cursor">
    Required for every teammate. Two parts:

    1. **Cursor integration** — Cursor Settings → Integrations → Linear (OAuth)
    2. **Linear MCP** — set `LINEAR_API_KEY` in your environment; toggle **linear** on under Settings → MCP

    Default motion: agents file Linear issues when work is identified. See [Workflow](/concept/workflow).
  </Step>
  <Step title="Run the docs site">
    From the docs repo root:

    ```bash
    mint dev
    ```

    Preview at `http://localhost:3000`.
  </Step>
  <Step title="First contribution">
    Read a spec before you write code. For features in **mono**, the loop starts with `/speckit.specify` — see [Workflow](/concept/workflow). For docs changes here, edit MDX and run `mint validate` before opening a PR.
  </Step>
</Steps>
