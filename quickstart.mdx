Everything from a blank Mac to the app running locally. Follow top to bottom, each step tells you what success looks like.

## Install the tools

<Steps>
  <Step title="Homebrew">
    Paste in your terminal ([docs.brew.sh](https://docs.brew.sh/Installation)):

    ```bash
    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
    ```

    Success: `brew --version` prints a version number.
  </Step>
  <Step title="Toolbelt">
    One command installs everything:

    ```bash
    brew install gh node oven-sh/bun/bun planetscale/tap/pscale
    ```

    That is GitHub CLI, Node, Bun, and the PlanetScale CLI.
  </Step>
  <Step title="Sign in">
    ```bash
    gh auth login
    pscale auth login
    ```

    Each opens a browser, approve, done. Also install [Cursor](https://cursor.com) if you have not.
  </Step>
</Steps>

## Run the app

<Steps>
  <Step title="Clone">
    ```bash
    gh repo clone bonafiai/mono
    gh repo clone bonafiai/docs
    ```

    **mono** is the application, **docs** is this site.
  </Step>
  <Step title="Configure">
    ```bash
    cd mono
    bun install
    cp .env.example .env.local
    ```

    Fill the required values in `.env.local`.
  </Step>
  <Step title="Database">
    Your own dev branch, real PlanetScale Postgres, mirrors prod:

    ```bash
    pscale branch switch <your-name> --database bonafi --create
    bun run db:migrate
    bun run dev:seed
    ```

    Point `DATABASE_URL` in `.env.local` at the branch connection string. Offline fallback: `bun run dev:db:up` starts a local container.
  </Step>
  <Step title="Run">
    ```bash
    bun run dev
    ```

    Success: the app on `http://localhost:3000` and the Inngest dev server running next to it.
  </Step>
</Steps>

## Connect Cursor

Open the repo in Cursor. The MCP servers (Mintlify, Linear) are preconfigured in `.cursor/mcp.json`, Cursor lists them automatically. When prompted, approve the Linear OAuth in your browser, that is the whole setup. Agents can now create and update issues, see [Workflow](/method/workflow).

## Run the docs site

```bash
cd docs
npx mint dev
```

Success: this site on `http://localhost:3000`. Validate changes with `npx mint validate`.
