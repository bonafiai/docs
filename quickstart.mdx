Get the app running locally, then the docs site. No theory required.

## Prerequisites

- [Cursor](https://cursor.com)
- [GitHub CLI](https://cli.github.com) (`gh auth login`)
- [Bun](https://bun.sh) 1.2+
- [PlanetScale CLI](https://planetscale.com/docs/concepts/planetscale-environment-setup) (`pscale auth login`)

## Run the app

<Steps>
  <Step title="Clone">
    ```bash
    gh repo clone bonafiai/mono
    gh repo clone bonafiai/docs
    ```

    **mono** is the application, **docs** is this spec hub.
  </Step>
  <Step title="Install and configure">
    ```bash
    cd mono
    bun install
    cp .env.example .env.local
    ```

    Fill the required values in `.env.local`.
  </Step>
  <Step title="Dev database">
    Create your own PlanetScale dev branch, one branch per developer:

    ```bash
    pscale branch switch <your-name> --database bonafi --create
    bun run db:migrate
    bun run dev:seed
    ```

    Point `DATABASE_URL` in `.env.local` at the branch connection string. It is real PlanetScale Postgres 17, so dev mirrors prod exactly. Offline fallback: `bun run dev:db:up` starts a local container.
  </Step>
  <Step title="Run">
    ```bash
    bun run dev
    ```

    Starts the app and the Inngest dev server together.
  </Step>
</Steps>

## Connect Cursor

<Steps>
  <Step title="Open in Cursor">
    Open the repo. Tooling provisions itself from `.cursor/mcp.json`, `skills-lock.json`, and `AGENTS.md`.
  </Step>
  <Step title="Authenticate Linear">
    The Linear MCP server is preconfigured. Approve the OAuth prompt in your browser when Cursor asks, that is all. Agents create and update issues from day one, see [Workflow](/concept/workflow).
  </Step>
</Steps>

## Run the docs site

```bash
cd docs
npx mint dev
```

Preview at `http://localhost:3000`. Validate before a PR with `npx mint validate`.
