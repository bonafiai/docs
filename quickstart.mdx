## Install the tools

<Steps>
  <Step title="Homebrew">
    Paste in your terminal ([docs.brew.sh](https://docs.brew.sh/Installation)):

    ```bash
    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
    ```

    Success: `brew --version` prints a version number.
  </Step>
  <Step title="Toolbelt">
    One command installs everything:

    ```bash
    brew install gh node oven-sh/bun/bun planetscale/tap/pscale
    ```

    That is GitHub CLI, Node, Bun, and the PlanetScale CLI.
  </Step>
  <Step title="Sign in">
    ```bash
    gh auth login
    pscale auth login
    ```

    Each opens a browser, approve, done. Also install [Cursor](https://cursor.com) if you have not.
  </Step>
</Steps>

## Run the app

<Steps>
  <Step title="Clone">
    ```bash
    gh repo clone bonafiai/web
    gh repo clone bonafiai/docs
    ```

    **web** is the application, **docs** is this site. Open both in one Cursor window (`~/www` with `docs/` and `web/` siblings) or open `web/` alone for feature work.
  </Step>
  <Step title="Configure">
    ```bash
    cd web
    bun install
    cp .env.example .env.local
    ```

    Fill WorkOS and Inngest keys when you have them. The scaffold runs without a database until the first persistence spec lands.
  </Step>
  <Step title="Database">
    When a spec adds tables, use your own PlanetScale Postgres branch:

    ```bash
    pscale branch switch <your-name> --database bonafi --create
    ```

    Point `DATABASE_URL` in `.env.local` at the branch connection string, then apply migrations:

    ```bash
    bun run db:migrate
    ```

    See [Database](/architecture/database) for the generate and migrate loop. Until the repo has migrations, skip this step.
  </Step>
  <Step title="Run">
    ```bash
    bun run dev
    ```

    Success: the microfrontends proxy on `http://localhost:3024`, product app at `/`, test app at `/test`.
  </Step>
</Steps>

## Connect Cursor

Open `web/` in Cursor. MCP servers are in `.cursor/mcp.json`; approve Linear OAuth when prompted. Spec-kit `/speckit.*` commands ship in the repo. Read `.specify/memory/constitution.md` before feature work.

## Run the docs site

```bash
cd docs
npx mint dev
```

Success: this site on `http://localhost:3000`. Validate changes with `npx mint validate`.
