title: "Rules"
description: "Hard invariants every implementation satisfies."
---

Rules are tool-agnostic. They state what is always true, not which vendor provides it. Vendor choices live in [Overview](/stack/overview).

In the code repo, spec-kit reads these from `.specify/memory/constitution.md`, which mirrors this page. See [Workflow](/concept/workflow).

1. **One tenant key everywhere.** Session claim = database foreign key = permission object = connection scope.

2. **Explicit tenant predicates on every query.** Tenant-scoped work runs inside a tenant-bound transaction. Row-level security is defense-in-depth, never the only guard.

3. **One async backbone.** Everything slow flows through a single typed publish contract into a durable bus. No slow work in request handlers. Jobs are idempotent or concurrency-keyed.

4. **One model gateway.** All LLM calls go through one registry and one gateway. No provider SDK at call sites.

5. **One object store.** Content-addressed keys. Objects are immutable once written.

6. **One parser seam.** One frozen parse artifact per source. The parser vendor is swappable behind the seam.

7. **One big JSONB.** Exactly one in the system: the raw parse artifact. Everything else is relational.

8. **Truth through commit only.** Approved proposals become truth in one transaction with their decision record.

9. **Replay, never mutate.** Re-running a stage creates a new run over the same input. Nothing is overwritten in place.

10. **Re-verify authorization.** The edge gate is optimistic. Every handler re-checks identity and permissions.

11. **Tooling is versioned in git.** MCP servers, skills, and agent rules live in committed config, never ad-hoc local setup.

12. **No feature without a spec.** No spec without acceptance scenarios. No merge while a scenario's E2E test fails.
